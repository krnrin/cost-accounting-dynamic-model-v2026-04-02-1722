## 三、核心文件 ② — 吉利E281报价核算.xlsx（17 sheets）

结构与定点版完全一致（少了 2 个 sheet：成本分析与追踪、客户目标价-初始），**但数据有一处关键差异**：

### ⚠️ 定点版 vs 报价版 关键差异

| 维度 | 定点版 | 报价版 | 差异说明 |
|------|--------|--------|----------|
| **6608516992 材料成本** | **81.26** | **64.26** | 差 ¥17，报价版更低 |
| 6608516992 出厂价 | 139.87 | 120.65 | 同上导致出厂价差 ¥19 |
| 6608516992 到厂价 | 142.24 | 123.02 | 差 ¥19 |
| **铜重单位** | **kg**（0.022561） | **g**（22.561） | ⚠️ 报价版铜重放大了 1000 倍 |
| 铝重单位 | kg（两版一致） | kg（两版一致） | 铝重无差异 |
| 其他 10 条线束 | 一致 | 一致 | 只有 6608516992 有数据差异 |

### 🔴 重大发现：铜重单位不一致

- **定点版**：铜重 = 0.022561（kg/根），如直流母线 6608491523
- **报价版**：铜重 = 22.561（g/根），即放大 1000 倍
- **铝重**：两版均为 kg 单位（0.2394392），无差异
- **结论**：定点版铜重已经是 kg，报价版铜重是 g。当前种子数据取自定点版（kg），是正确的

## 八、与种子数据（_e281_harness_seed_data.json）的比对

### ✅ 一致项

- 内部费率 — 完全一致（labor=29.19, 6D-MOH 合计=19.74）

### ⚠️ 潜在问题

#### 1. BOM 简化为单行
种子数据将每条线束的 BOM 简化为 **1 行**（整根线束的总材料成本作为 `unitPrice`），而实际 Excel BOM 有 **150-220 行**散件明细。
- 这是因为核算引擎目前只做"线束级"汇总计算（L2 精度），尚未实现行级 BOM（L3 精度）
- 简化方式：`unitPrice = materialCost`，`qty = 1`，铜铝重取汇总值
- 结果：在 unitPrice-first 策略下计算正确

#### 2. 包装费结构差异
种子数据中的包装费拆分：
```
packaging: { innerBoxCost, outerBoxCost, palletCost, trayDividerCost, bubbleWrapCost, labelCost, subtotal }
freight: { freight, excessFreight, shortHaul, thirdPartyWarehouse, storage, subtotal }
```

Excel 中的实际拆分：
```
内包装（含隔板+气泡膜+标签等合计） | 外包装 | 运费 | 超额运费 | 短驳 | 三方仓 | 仓储
```

种子数据把 `innerBoxCost` 设为了 Excel 的 `内包装` 值（已含隔板+气泡膜），而 `trayDividerCost`/`bubbleWrapCost`/`labelCost` 设为 0。这意味着细分维度没有还原，但总额是对的。

#### 3. 定点版 vs 报价版选择
种子数据标注 `source: 吉利E281定点核算.xlsx`（定点版），数据验证也与定点版一致。这是正确的选择，因为定点版代表最终确认价格。
