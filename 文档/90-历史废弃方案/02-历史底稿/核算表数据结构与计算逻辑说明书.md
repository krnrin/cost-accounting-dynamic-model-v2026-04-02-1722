## 2. 核心计算链路——成本金字塔

从一根高压线束的最终到厂价（客户支付价）层层分解：

```
到厂价 ¥138.26（以 6608491523 为例）
│
├── 出厂价 ¥133.79
│   │
│   ├── 材料成本 ¥88.07  ←── KSK线束BOM明细逐行汇总
│   │   │
│   │   ├── 非导线物料（连接器、端子、护套、胶带等）──→ 二次物料明细 VLOOKUP 单价
│   │   └── 导线物料（铜包铝导线、屏蔽线等）         ──→ 铜铝重量 × 金属价格 + 非金属部分
│   │
│   ├── 废品率成本 ¥0.88  = 材料成本 × 1.0%
│   │
│   ├── 直接人工 ¥13.09   = 实际工时 × 35 元/H
│   │
│   ├── 制造费 ¥17.46     = 实际工时 × 46.69 元/H
│   │
│   ├── 管理费 ¥7.12      = (材料 + 人工 + 制造) × 6%（不含废品）
│   │
│   └── 利润 ¥7.17        = (材料 + 废品 + 人工 + 制造 + 管理) × 5.6627%
│
├── 包装费 ¥2.29  = 内包装 + 外包装
│
└── 运输费 ¥2.18  = 短驳 + 三方仓 + 仓储
```

**单车成本** = Σ(每条线束到厂价 × 装车比) = **¥526.63**

### 3.2 导线单价计算公式（核心）

```
导线单价(元/单位) = 铝重(g) ÷ 1000 × 18.91 + 铜重(g) ÷ 1000 × 76.45 + 非铜(元) ÷ 1000
```

其中：
- **18.91** = 铝价（元/kg）= 18910 元/吨 ÷ 1000
- **76.45** = 铜价（元/kg）= 76450 元/吨 ÷ 1000
- **非铜** = 导线中除铜铝外的成本（绝缘层、编织层等）

⚠️ **重要**：金属价格硬编码在公式中（铜=76450元/吨，铝=18910元/吨），这就是我们引擎中 metalPrices 参数的来源。

### 3.3 材料成本汇总公式

**每行 BOM 的材料成本** = (非导线单价 + 导线单价) × 数量

**每条线束的总材料成本** = Σ 所有 BOM 行的材料成本

### 5.3 公式验证结果（11条线束全部通过）

通过对每条线束逆向验算，确认了以下精确的费率和基数：

#### 废品率

```
废品率成本 = 材料成本 × 1%（Excel 公式硬编码 =G*1%）
```

- 表头标注"0.9%"，Excel 公式实际写的是 `1%`
- **本质说明**：废品率是**吉利客户模板的要求**，由客户指定。这个比率在客户报价中具体填多少，由销售根据谈判策略自行调整，不需要在模型中深究
- **内部核算中的废品/损耗**：已在"运营工时费报价基准"中以 **0.5% 材料损耗** 的形式计入内部 6 维制造费，与客户报价的废品率是两套独立体系

#### 人工费率 / 制造费率 / 管理费率 / 利润率

```
直接人工 = 实际工时 × 35.00 元/H
制造费 = 实际工时 × 46.69 元/H（表头写47，实际46.69）
管理费 = (材料 + 人工 + 制造) × 6%（基数不含废品）
利润 = (材料 + 废品 + 人工 + 制造 + 管理) × 5.6627%
```

> **重要提示**：以上费率的具体数值（35/46.69/6%/5.6627%）同样属于**销售报价策略层面**，销售会根据项目实际情况调整。核算模型不需要关注这些中间费率的精确含义——**报价单中真正有用的数据只有出厂价和到厂价**，以及分摊费用（工装/试验分摊的回收需放到项目跟踪项中管理）。

#### 出厂价

```
出厂价 = 材料成本 + 废品率成本 + 直接人工 + 制造费 + 管理费 + 利润
```

✅ 验证：差异 < 1e-10（浮点精度级别）。

#### 到厂价

```
到厂价 = 出厂价 + 包装费 + 运输费
```

✅ 验证：差异 < 1e-10。

### 5.4 汇总行（Row 17）

```
单车成本 = Σ(到厂价ᵢ × 装车比ᵢ)，i = 1..11
         = 526.631432587297（¥526.63）
```

验证：装车比加权求和完全匹配。

Row 19 还有一个值 ¥479.95，这是 **客户目标价加权合计**（不含分摊）。

### 6.1 双轨费率体系

| 费率项目 | 客户报价费率 | 内部运营费率(昆山K1K2, 90%效率) |
|----------|------------|-------------------------------|
| 直接人工费率 | **35.00 元/H** | **29.19 元/H** |
| 间接人工 | （不单独列示） | 6.88 元/H |
| 低值易耗品 | （不单独列示） | 0.95 元/H |
| 物料消耗 | （不单独列示） | 0.68 元/H |
| 厂房分摊 | （不单独列示） | 5.26 元/H |
| 自动化分摊 | （不单独列示） | 1.72 元/H |
| 其他制造费 | （不单独列示） | 4.25 元/H |

## 8. Sheet 6：配置明细——装车比矩阵

### 8.1 作用

一个车型有多种配置（如不同电池版本、不同驱动形式），每种配置使用不同的线束组合。**装车比(用量)** = 该线束在所有配置中的加权平均使用概率。

### 8.2 配置矩阵结构

| | 配置1 | 配置2 | ... | 配置N | 加权装车比 |
|---|---|---|---|---|---|
| 6608491523 | 1 | 0 | ... | 1 | 0.525 |
| 6608491524 | 0 | 1 | ... | 0 | 0.105 |
| ... | | | | | |

每个配置的 take rate（市场占比预测）× 该配置是否使用该线束 = 最终装车比。

11条线束的装车比之和不等于1，因为：
- 直流母线是互斥的（4选1）：0.525 + 0.105 + 0.07 + 0.105 = 0.805
- 电动压缩机是互斥的（4选1）：0.595 + 0.105 + 0.03 + 0.225 = 0.955
- 充电插座是互斥的（3选1）：0.525 + 0.105 + 0.07 = 0.70

这些小于1的比例说明还有其他配置未列入（或有低压替代方案）。

## 14. 关键发现与注意事项

### 14.1 报价中间费率属于销售策略范畴

客户报价逻辑中的各项费率（废品率、人工费率、制造费率、管理费率、利润率）是**销售根据谈判策略设定**的，不代表真实内部成本结构：

| 字段 | Excel 公式值 | 备注 |
|------|-------------|------|
| 废品率 | 1%（公式硬编码） | 吉利模板要求，销售可调 |
| 人工费率 | 35 元/H | 高于内部实际 29.19 |
| 制造费率 | 46.69 元/H | 高于内部实际 19.74 |
| 管理费 | 6%（基数不含废品） | |
| 利润率 | 5.6627% | |

**核算模型真正关注的输出是出厂价和到厂价**，中间费率拆分只是报价呈现格式。

### 14.2 内部材料损耗 vs 客户废品率

两套独立体系，不要混淆：
- **内部实绩**：材料损耗 0.5%，计入运营工时费报价基准的 6 维制造费中
- **客户报价**：废品率由客户模板指定（吉利模板写 0.9%），Excel 公式中实际用 1%，具体多少由销售决定

### 14.4 铜铝价格硬编码

BOM 公式中金属价格硬编码：
- 铜价 = 76.45 元/kg（= 76450 元/吨）
- 铝价 = 18.91 元/kg（= 18910 元/吨）

如果金属价格变动，需要修改 BOM 公式或使用引擎的 metalPrices 参数重新计算。

### 14.5 实际工时的来源

实际工时是外部输入（工艺部门提供），不是从 BOM 计算得出的。工时数据精确到小数点后15位（如 `0.374029065686274`），说明是从更精细的工序工时汇总而来（前工程+后工程，但在报价逻辑中只显示合计值）。

### 14.6 包装物流费运费为零

11条线束的"运费"和"超额运费"列全部为 0。实际运输成本体现在"短驳""三方仓""仓储"三个子项中。这说明主运费已包含在出厂价或由客户承担。

### 14.7 6608516992 定点 vs 报价差异

## 15. 一次性费用分摊：两套逻辑的差异分析

> **与系统设计的对应关系补充**：
> - 本文档解释的是“财务核算表 / 定点核算工作簿”中的现实结构与公式来源。
> - 面向系统建模时，BOM 行、吉利报价单、一次性费用分摊不应继续只以 Excel sheet 或列号理解，而应分别落到独立业务对象：BOM 行对象、报价快照对象、分摊项对象、回收跟踪对象。
> - 因此，本文档中的 Excel 结构是系统设计依据之一，但不应直接等价为系统内部对象结构。

> **核心结论**：一次性费用分摊回收在财务测算口径与销售 / 客户执行口径之间存在表达差异。系统落地时，应以实际业务执行口径为主，同时保留财务口径的对照与解释能力。

### 15.2 财务核算表的分摊逻辑（简化算法）

**数据来源**：`BOM核对/吉利E281定点核算.xlsx` → 项目评估汇总（昆山90%）sheet

财务核算表采用**项目级打包分摊**：

```
项目评估汇总 Row 35:
  I35 = 项目工装投入!L21 = ¥418,000 ÷ 50,000 = ¥8.36/套

项目工装投入 sheet:
  L20 = 客户目标价!H13 = ¥418,000（客户确认的工装总额）
  L21 = L20 ÷ 50,000 = ¥8.36（按套分摊）

项目评估汇总 Row 36:
  研发费用分摊 = ¥131.32万 ÷ 5年 = ¥26.264万/年

项目评估汇总 Row 37:
  客户目标不含分摊价 = ¥479.25/车
```

**问题**：
- 把全项目工装总额 ¥418,000 笼统除以 50,000 **套**，得到 ¥8.36/套
- 没有区分哪条线束有工装投入、哪条没有
- 研发费 ¥131.32万 也参与了分摊计算，但吉利模板中研发费 = 0

### 15.3 两套算法的结构性差异

| 维度 | 吉利模板（按根独立分摊） | 财务核算表（按套笼统分摊） |
|------|----------------------|------------------------|
| **分摊单位** | 根（每条线束号独立） | 套（全项目打包） |
| **分摊基数** | 50,000 根/线束号 | 50,000 套/项目 |
| **参与分摊的线束** | 7条（4条无费用不参与） | 全部11条统一 ¥8.36 |
| **分摊单价** | 各不相同（¥0 ~ ¥6.46） | 统一 ¥8.36/套 |
| **研发费** | 不体现（= 0） | ¥131.32万 ÷ 5年 参与计算 |
| **试验费** | 按线束独立（¥0 ~ ¥5.50） | 未单独体现 |
| **精确度** | 精确到线束号级 | 项目级粗算 |

由于每条线束的装车比（vehicle ratio）不同，**实际年产量差异巨大**，导致各线束达到 50,000 根分摊基数的时间完全不同：

**关键洞察**：
- 装车比 ≥ 0.5 的线束（6条）：不到1年就分摊完5万根，**分摊回收完毕后应降价**（去掉分摊部分）
- 装车比 0.07 的 6608544875：需7.14年，**超出5年项目周期，分摊无法全部回收**
- 装车比 0.03 的线束（如有分摊）：需16.7年，**远超项目周期**

这就是为什么需要**分摊回收跟踪功能**——跟踪每条线束的累计产量，当达到分摊基数时提醒销售调价。

### 15.5 程序应采用的分摊算法

**结论：程序必须按吉利模板的"按根/线束号独立分摊"算法实现，不使用财务核算表的"按套笼统分摊"。**

程序中的一次性费用分摊数据模型：

```typescript
interface OnetimeCostAllocation {
  harnessId: string;           // 线束号
  toolingTotal: number;        // 总工装费（¥）
  testingTotal: number;        // 总试验费（¥）
  rdTotal: number;             // 总研发费（¥，通常=0，不向客户回收）
  allocBase: number;           // 分摊基数（根，通常=50000）
  toolingPerUnit: number;      // 工装摊/件 = toolingTotal ÷ allocBase
  testingPerUnit: number;      // 试验摊/件 = testingTotal ÷ allocBase
  rdPerUnit: number;           // 研发摊/件 = rdTotal ÷ allocBase
  totalPerUnit: number;        // 合计分摊/件
  participatesInAlloc: boolean; // 是否参与分摊
}

// 分摊回收跟踪
interface AllocRecoveryTracker {
  harnessId: string;
  allocBase: number;           // 分摊基数（根）
  vehicleRatio: number;        // 装车比
  annualVolume: number;        // 年产量（根）= 装车比 × 年产能
  yearsToRecover: number;      // 回收所需年数 = allocBase ÷ annualVolume
  cumProduced: number;         // 累计已生产（根）
  isRecovered: boolean;        // 是否已回收完毕
  recoveredDate?: string;      // 回收完毕日期
  priceAdjustmentNeeded: boolean; // 是否需要调价
}
```

**到厂价公式（含分摊）**：
```
到厂价 = 出厂价 + 包装费 + 运输费 + 工装摊/件 + 试验摊/件
```

**单车成本公式（含分摊）**：
```
单车成本(含分摊) = Σ(到厂价_含分摊 × 装车比)
单车成本(不含分摊) = Σ(到厂价_不含分摊 × 装车比)
```
