# 测试验收清单-P0主链路

> 适用范围：高压线束精算与决策引擎 P0 主链路  
> 来源依据：统一 PRD、研发实施清单、对应功能规格与开发任务书  
> 目标：覆盖当前阶段必须跑通的主链路验收项

- 项目管理与Dashboard

## 2. 测试执行原则
- 先验证对象与入口存在
- 再验证上下文切换稳定
- 再验证模块间基础联通
- 当前 `app/` 实际行为为主依据

---

## 3. 项目管理与Dashboard

### A 级必验
- 能打开项目列表页
- 能看到项目列表数据
- 能按项目名 / 编号 / 客户搜索
- 能按状态筛选
- 能执行新建 / 导入 / 删除入口
- 能进入项目 Dashboard
- Dashboard 中能看到关键摘要与主模块入口

### B 级增强验收
- 从项目列表进入 Dashboard 时项目上下文正确
- Dashboard 中模块入口跳转后仍保持项目上下文
- Dashboard 摘要与下游模块显示口径不明显冲突

### C 级增强验收
- 项目层锁定或治理状态
- 更复杂项目级汇总

---

## 4. 场景管理

### A 级必验
- 在项目下能看到场景入口
- 能看到多个场景或场景列表结构
- 场景类型可识别
- 能进入具体场景详情或上下文

### B 级增强验收
- 切换不同场景后上下文正确
- 不同场景的数据不互相覆盖
- 下游模块能识别当前场景
- 有基础比较入口或比较结构预留

### C 级增强验收
- 场景冻结
- 场景发布
- 场景继承

---

## 5. BOM工作簿

### A 级必验
- 能进入 BOM 页面
- 能看到逐行 BOM 数据
- 能按线束号筛选
- 能按分类筛选
- 能看到基础汇总信息

### B 级增强验收
- BOM 与当前场景上下文一致
- 下游报价 / 设变读取的是同一 BOM 口径
- 行级与线束号级汇总口径一致

### C 级增强验收
- 版本差异比较
- 深化导入校验

---

## 6. 客户报价工作台

### A 级必验
- 能进入报价页面
- 能看到客户报价结果
- 能看到内部成本基线
- 能看到差异值或利润空间
- 能在项目 / 场景 / 线束号上下文中查看结果

### B 级增强验收
- 场景切换后报价上下文同步切换
- 参数快照或等价结构存在
- 差异结果与内部成本基线口径一致

### C 级增强验收
- 模板输出深化
- 参数审批治理
- 正式发布状态

---

## 7. P0 跨模块联调必验

### A 级必验
- 从项目列表进入 Dashboard
- 从 Dashboard 进入场景上下文
- 从场景进入 BOM 与报价
- 当前项目 / 场景上下文在链路中不丢失

### B 级增强验收
- BOM → 报价的基础数据链路可解释
- 场景变化能影响 BOM / 报价上下文
- 摘要页与专业页口径基本一致

---

## 8. 阻塞判定

以下任一情况出现，P0 不应视为通过：
- 项目上下文丢失
- 场景上下文丢失
- BOM 数据与报价结果明显不在同一上下文
- 主入口链路无法连续走通

---

## 9. 结论

P0 主链路验收的核心，不是某一页是否存在，而是系统是否已经形成“项目 → 场景 → BOM / 报价”的稳定工作主链。如果这条主链不能稳定跑通，后续执行闭环与治理能力都缺少基础。
