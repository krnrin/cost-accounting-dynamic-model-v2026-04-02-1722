## 2. 测试执行原则
- 先验证基础治理结构是否存在
- 再验证是否真正支撑主链路结果
- 再验证是否形成管理和产品化增强效果
- 不以页面多少判断通过，而以结构与口径是否成立判断

---

## 3. 版本与发布治理

### A 级必验
- 关键对象存在版本结构或等价关系
- 版本状态存在
- 当前有效版本可识别
- Trace ID 或等价唯一标识存在

### B 级增强验收
- 场景 / BOM / 参数 / 报价可挂接版本
- 状态切换可追溯
- 可支撑后续发布流深化

### C 级增强验收
- 完整审批流
- 审计专题页
- 复杂版本比较

---

## 4. 系统设置与参数治理

### A 级必验
- 现有设置项可稳定维护
- 参数分类清晰
- 关键参数可被主链路模块引用

### B 级增强验收
- 全局 / 项目 / 场景作用域区分明确
- 关键参数具备快照或等价引用结构
- 参数调整影响范围正确

### C 级增强验收
- 审批发布
- 精细权限治理
- 深度审计

---

## 5. Simulation与年降管理

### A 级必验
- 能调整关键参数并看到变化结果
- 能查看年降参数与影响结果

### B 级增强验收
- 仿真与正式结果分层清晰
- 年降能挂接场景或版本
- 基线引用明确

### C 级增强验收
- 一键转场景
- 更重周期分析
- 治理深化

---

## 6. 管理决策舱

### A 级必验
- 能查看关键管理摘要
- 能从摘要跳转到专业模块

### B 级增强验收
- 成本 / 报价 / 分摊 / 跟踪 / 预警摘要能稳定汇总
- 差异与异常能优先暴露

### C 级增强验收
- 跨项目对比
- 治理摘要
- 更复杂管理驾驶舱能力

---

## 7. Profile与个人中心

### A 级必验
- 有页面入口
- 能查看个人基础信息
- 能保存个人偏好

### B 级增强验收
- 能与主题或界面偏好联动
- 不影响主业务口径参数

### C 级增强验收
- 权限概览
- 通知概览
- 更重的个人工作台扩展

---

## 8. P2 跨模块联调必验

### A 级必验
- 版本与参数治理能力能支撑主链路对象
- 管理决策舱能读取真实摘要来源

### B 级增强验收
- 仿真、年降、版本、参数之间引用关系正确
- 管理决策舱摘要与专业模块口径一致
- Profile 偏好不干扰业务结果

---

## 9. 阻塞判定

以下任一情况出现，P2 不应视为通过：
- 有版本字段但无有效版本判断
- 参数有页面但无稳定引用来源
- 仿真结果与正式结果混用
- 决策舱摘要与专业模块明显冲突
- Profile 改动影响业务计算结果

---

## 10. 结论

P2 治理与增强验收的核心，不是堆更多管理页面，而是确认系统是否已经从“会算、会跟踪”继续升级为“可治理、可追溯、可决策、可产品化使用”。这层通过后，系统才更接近真正的平台化产品形态。
