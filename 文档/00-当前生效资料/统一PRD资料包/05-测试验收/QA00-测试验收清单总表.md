# 高压线束精算与决策引擎测试验收清单总表

> 目标：把统一 PRD、专项规格与开发任务书中的验收建议整理为统一测试验收入口  
> 使用对象：测试、产品、开发联调、阶段验收  
> 使用原则：按 A / B / C 三级验收执行，A 为当前阶段必验，B 为闭环增强验收，C 为后续增强或待拍板能力验收

---

## 1. 验收分级说明

### A 级
当前阶段必验项。通常对应：
- 页面或入口存在
- 基础对象存在
- 基础状态存在
- 主链路可跑通

### B 级
闭环增强验收项。通常对应：
- 模块联动
- 状态流稳定
- 摘要和来源一致
- 可比较、可追溯、可挂接

### C 级
增强能力或待拍板项。通常对应：
- 审批
- 高级治理
- 深度比较
- 外部通知
- 产品化增强项

## 4. 验收执行建议

### 4.1 先单模块，再跨模块
- 先验单模块 A 级
- 再验模块间 B 级联动
- 最后看 C 级增强能力

### 4.2 先对象和状态，再看页面细节
- 先确认对象存在
- 再确认状态存在
- 再确认页面承载与交互

### 4.3 先以当前代码为准
- 历史会话和历史截图只能作为补充证据
- 当前主应用 `app/` 代码和实际联调结果是验收主依据
