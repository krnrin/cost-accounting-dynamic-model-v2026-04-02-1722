## 1. 文档目标

本文档以真实吉利总报价模板为参照，但站在系统边界内说明其可参考部分。目标是说明：
- 吉利最终客户报价单在真实模板中由哪些 sheet 组成

## 2. 吉利报价单在真实模板中的位置

真实吉利报价单对应文件：

`高压线束包1-总报价模板(新)-20251117132314103 - 定点.xls`

当前模板包含 6 个 sheet：

1. `填写说明`
2. `价格汇总`
3. `4.1工装费`
4. `4.2.试验费`
5. `4.3.研发费`
6. `工装分类&工序内容&工序号对应关系（非报价内容）`

其中真正构成“吉利报价单”的核心，是：
- `价格汇总`
- `4.1工装费`
- `4.2.试验费`
- `4.3.研发费`

但这份报价单是销售最终按商务策略调整后、提交到客户系统并被客户承认的结果文件。它不应被系统内核直接建模为正式可计算对象，更适合作为**内部核算结果的对照参照物**，重点参考其中的分摊费用及回收方式、出厂价、到厂价和定点确认口径。

---

## 3. 系统应参考的对象层级

### 3.1 吉利最终报价单对象（Reference Workbook）
承载销售最终提交给客户并被客户承认的一整套报价模板。

### 3.2 价格汇总参考行（Reference Price Summary Row）
`价格汇总` 按线束零件号逐行展示最终客户口径结果，系统主要参考其中的：
- 出厂价（不含分摊）
- 总到厂价（未税）
- 总投资费用

### 3.3 工装费参考明细行（Reference Tooling Fee Row）
`4.1工装费` 用于识别客户最终认可的工装费用表达方式。

### 3.4 试验费参考明细行（Reference Test Fee Row）
`4.2.试验费` 用于识别客户最终认可的试验费用表达方式。

### 3.5 研发费参考明细行（Reference R&D Fee Row）
`4.3.研发费` 用于识别客户最终认可的研发费用表达方式。

### 3.6 工装分类映射表（Tooling Mapping Reference）
最后一个 sheet 不是报价结果本身，而是工装分类与工序号的辅助规则表。

## 5. 真实字段语义

## 5.1 价格汇总页是客户最终报价主表

`价格汇总` 的核心口径非常明确：

- **总到厂价（未税）** = 客户最终看到的总价口径
- **出厂价（不含分摊）** = 直接材料 + 制造费用 + 期间费用及利润
- **1.直接材料** 下又拆为：
  - 原材料成本（A1）
  - 外购件成本（A2）
- **2.制造费用** 下至少包括：
  - 加工费用（B1）
  - 废品损失（B2）
- **3.期间费用+利润** 下至少包括：
  - 管理费用（C1）
  - 财务费用（C2）
  - 销售费用（C3）
  - 利润（D）
- **4.总投资费用** 当前模板首列展示为总工装费用

这说明吉利模板的真实报价核心不是此前写的“铜重、铝重、工时、包装费、运输费”那套结构，而是**按客户模板要求分成直接材料、制造费用、期间费用+利润、总投资费用**四大块。

## 5.2 工装费不是一个简单汇总数

`4.1工装费` 不是只给总额，而是展开到：
- 工装分类
- 子零部件编号/名称
- 新开/借用
- 工序内容
- 工序号
- 工装名称
- 模穴数
- 寿命
- 外形尺寸
- 设计费用
- 上模/下模品牌与费用

所以系统里不能只保留 `totalToolingFee` 一个数字，至少要保留工装明细层。

## 5.3 试验费按项目逐项展开

`4.2.试验费` 的总试验费用公式为：

```text
总试验费用 F = (每次试验费 A + 样件数量 C × 样件价格 D + 试验工装费用 E) × 试验次数 B
```

这说明试验费对象本身就是**结构化计算对象**，不是备注字段。

## 5.4 研发费是多费用篮子相加

`4.3.研发费` 的总研发费用公式为：

```text
总研发费用 P = A + B + C + D + E + F
```

其中：
- A = 硬件开发费用
- B = 软件开发费用
- C = 标定测试费用
- D = 开发工具费用
- E = 认证费用
- F = 差旅费

所以研发费不是单列金额，而是由多种研发活动汇总而成。

## 6. 真实计算链路

## 6.1 价格汇总主链路

模板中已经直接给出主公式定义：

```text
出厂价（不含分摊） P1 = A1 + A2 + B1 + B2 + C1 + C2 + C3 + D
总到厂价（未税） P2 = P1 + E1 + E2 + F1 + F2 + G1 + G2 + J1 + J2 + K + L
```

其中当前可直接确认的展示层为：
- A1 原材料成本
- A2 外购件成本
- B1 加工费用
- B2 废品损失
- C1 管理费用
- C2 财务费用
- C3 销售费用
- D 利润
- 4.总投资费用中至少有工装费

说明模板本身保留了更完整的客户报价分项，只是当前截取区未完全展开 E/F/G/J/K/L 各项中文标题。

## 6.2 工装费链路

```text
工装费明细
→ 按工装分类/工序/子零部件录入
→ 形成线束号级工装费用
→ 汇入价格汇总中的总投资费用
```

## 6.3 试验费链路

```text
试验项目明细
→ 计算单项总试验费用
→ 按零件号汇总
→ 进入总到厂价相关投资/附加费用口径
```

## 6.4 研发费链路

```text
研发工作内容明细
→ 分别计算硬件/软件/标定/工具/认证/差旅
→ 汇总总研发费用
→ 进入报价总成本口径
```

## 7. 吉利报价单与内部核算表的边界

### 7.1 内部报价核算表才是系统主对象
像：
- `吉利E281报价核算_BOM核对版.xlsx`
- `吉利E281定点核算_BOM核对版.xlsx`

这类文件对应的才是系统真正要承载和管理的内部核算逻辑。

### 7.2 吉利最终报价单不是系统可完全映射对象
吉利最终报价单是销售按商务策略调整后的最终提交版本，其中很多费用表达和利润安排已经过商务处理，和内部核算不存在稳定一一映射关系。

因此系统不应尝试把整份吉利最终报价单完整反向建模为可计算内核对象。

### 7.3 系统只参考少数关键对照口径
对于吉利最终报价单，系统主要参考：
- 分摊费用及回收方式
- 出厂价
- 到厂价
- 定点确认含义

除此之外的商务调整项，不应强行纳入系统映射规则。

---

## 8. 推荐落地约束

### 8.1 系统以内部核算为主
DS02 用于帮助理解吉利最终报价单，不改变系统以内部核算为主对象的边界。

### 8.2 只保留关键对照口径
系统如需引用吉利最终报价单，应优先只保留：
- 出厂价
- 到厂价
- 分摊费用表达
- 回收方式表达
- 是否定点/已被客户承认

### 8.3 不要尝试反推销售商务策略
销售最终报价中的商务调整项不应被强行拆回系统规则，也不应要求与内部核算逐字段映射。

### 8.4 工装分类与工序号映射仍可作为辅助参考
它属于模板辅助规则，可用于理解费用结构，但不是系统主报价结果对象。

## 9. 当前应补充到系统说明书中的结论

1. 吉利最终客户报价单真实载体是 `高压线束包1-总报价模板(新)-20251117132314103 - 定点.xls`。
2. 这份报价单是销售最终按商务策略调整后提交给客户并被承认的结果，不是系统主计算对象。
3. 系统主对象仍然是内部核算表；吉利最终报价单主要用于参考分摊费用及回收方式、出厂价、到厂价和定点含义。
4. 吉利用户口径下的很多费用调整与商务策略，不能要求与内部核算逐字段稳定映射。
5. DS02 的作用应是划清“可参考口径”和“不可强映射口径”，而不是要求系统完整承载最终客户报价单。

---

## 10. 结论

吉利报价单应被视为销售最终提交给客户并被客户承认的结果文件。对系统而言，它的价值不在于被完整建模，而在于帮助确认客户最终认可的分摊费用及回收方式、出厂价、到厂价和定点口径。系统主对象仍应坚持内部核算结构，不应把销售最终报价策略反向硬映射进内核。
