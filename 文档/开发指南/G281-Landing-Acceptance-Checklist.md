# G281 项目程序落地验收清单 (Landing & Acceptance Checklist)

本文档由 **张滔滔 (流程负责人)** 定义，用于指导 G281 高压线束动态核算模型的最终集成与交付验收。

## 1. 身份与权限校验 (Role & RBAC)
- [ ] **身份锁定**：系统顶栏实时显示当前用户身份（总助、财务、销售、程序员、流程）。
- [ ] **权限隔离**：
    - [ ] 财务 (王强)：唯一拥有基准费率发布权。
    - [ ] 销售 (翁骏)：拥有商务报价映射权，禁止修改实绩参数。
    - [ ] 流程 (张滔滔)：拥有审批流节点配置与合规审计权。

## 2. 数据红线与内核逻辑 (Data Redlines & Core Engine)
- [ ] **损耗率强制对齐**：内部核算逻辑锁定为 **0.5% 材料损耗**，拒绝任何 0.9% 的硬编码进入实绩层。
- [ ] **六维 MOH 穿透**：制造费用必须包含间接人工、低值易耗、机物料、厂房、自动化、其他。
- [ ] **90% 效率基准**：系统默认以 90% 效率进行损益预演，支持动态滑块调节。
- [ ] **去硬编码验证**：全局搜索代码库，确认无 `0.9%`、`35` 等写死的报价参数。

## 3. 流程闸门 (Process Gates)
- [ ] **5 节点基准发布流**：`DRAFT -> PRE_CHECK -> REVIEW -> PENDING_SIGNOFF -> ACTIVE` 闭环。
- [ ] **Audit_Trace_ID**：每一个激活的费率版本必须自动生成唯一的流程审计追踪 ID。
- [ ] **异常关闭机制**：当成本超支时，决策舱必须跳红预警，且在销售发起 `Exception Closing` 并获批前，禁止导出正式报价。

## 4. UI/UX 工业感标准 (UI/UX Standards)
- [ ] **48px 极简顶栏**：标题、状态、参数 Popover 与 Univer 公式栏像素级合并。
- [ ] **95% 垂直空间**：Univer 表格沉浸式铺满，移除所有冗余边框与侧边栏。
- [ ] **深色决策舱材质**：背景使用 `Slate-950`，面板使用磨砂玻璃效果 (`backdrop-blur`)。
- [ ] **数据美学**：核算数字使用 `JetBrains Mono` 字体，支持 `Number Ticker` 平滑滚动动效。

## 5. 集成联动验证 (Integration & Linkage)
- [ ] **Finance -> Sales 触发流**：财务更新基准后，受影响项目的销售端自动触发“重核准”预警。
- [ ] **多工厂对标**：支持在 UI 一键切换 7 大基地，瀑布图秒级刷新损益对比。
- [ ] **版本追溯**：支持对比“项目锁定版”与“财务最新版”的利润差异。
